# Инструменты для улучшения генерации отчётов

Идеи для второго разработчика: что можно добавить поверх текущего MVP (GigaChat + локальные черновики + шаблоны руководителя).

## Уже есть в проекте

| Инструмент | Где |
|------------|-----|
| Типизированный system prompt | `report_prompts.dart` |
| Шаблоны руководителя по типу отчёта | `TemplateProvider` |
| Мультимодальность (фото → GigaChat Files API) | `GigaChatService._uploadImages` |
| Валидация и анти-injection | `InputValidator` |
| Автосохранение черновика | `DraftAutosave` + `upsertDraft` |
| Ручная правка перед отправкой | `CreateReportScreen` |
| Обратная связь при отклонении | `managerFeedback` |

---

## Рекомендуемые следующие шаги (по приоритету)

### 1. Голосовой ввод (высокий приоритет для поля)

**Зачем:** сотрудник диктует заметки без клавиатуры.

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| `speech_to_text` (on-device) | Оффлайн, без доп. API | Качество зависит от устройства |
| GigaChat / SaluteSpeech STT | Единый вендор | Нужна сеть, отдельный API |

**Интеграция:** кнопка 🎤 у поля «Сырые заметки» → текст в `_rawController` → автосохранение.

---

### 2. Диктофон с чанками (как Telegram voice)

Запись `.m4a` локально → при сети отправка в STT → вставка в черновик. Подходит для длинных осмотров.

---

### 3. Чек-лист полей по типу отчёта

**Зачем:** до генерации показать «не заполнено: время инцидента, ответственный».

- Модель `ReportFieldDefinition` per `ReportType`
- UI: `CheckboxListTile` / прогресс «7/10 полей»
- В system prompt: «Обязательные поля: …; если нет — не указано»

---

### 4. Двухпроходная генерация (quality gate)

1. **Проход A:** GigaChat извлекает структурированные факты в JSON (`ExtractedFacts`).
2. **Проход B:** сборка markdown только из JSON (меньше галлюцинаций).

Дороже по токенам, но стабильнее для метрик и инцидентов.

---

### 5. RAG по шаблонам и регламентам

- PDF/MD регламенты компании в `assets/regulations/`
- При генерации: top-k фрагментов в контекст (локальный поиск по ключевым словам или эмбеддинги через GigaChat Embeddings API)
- Шаблон руководителя остаётся приоритетным

---

### 6. OCR с фото (дополнение к vision)

Если на фото таблица/паспорт данных — `google_mlkit_text_recognition` или API OCR → сырой текст в промпт как «цитата с фото».

---

### 7. Геометка и время

| Данные | Пакет | В промпт |
|--------|-------|----------|
| GPS | `geolocator` | «Координаты осмотра: …» |
| Время | `DateTime.now()` | «Время создания черновика: …» |

Не выдумывать адрес — только координаты или «не определено».

---

### 8. Предпросмотр markdown

`flutter_markdown` под полем результата — рабочий видит, как отчёт выглядит у руководителя.

---

### 9. Diff «было → стало» при повторной генерации

После второй генерации — подсветка изменений (`diff_match_patch`), чтобы не потерять правки вручную.

---

### 10. Локальная очередь и фоновая синхронизация

Уже есть `send_queue_v1`. Расширить:

- `Workmanager` / foreground service: отправка черновиков и отчётов при появлении сети
- Retry с exponential backoff

---

### 11. Метрики качества (для хакатона)

Логировать (без PII): длина ввода, число фото, время ответа GigaChat, % отклонений руководителем — для A/B промптов.

---

### 12. Function calling / tools (GigaChat)

Если API поддерживает tools:

- `extract_incident_timeline(notes) -> JSON`
- `validate_metrics(notes) -> missing_fields[]`

Один вызов tool + финальный markdown.

---

## Что не стоит тянуть в MVP

- Firebase / Google ML Kit Cloud (ограничение Aurora)
- Полноценный CMS шаблонов на сервере (пока mock)
- Автоперевод — продукт русскоязычный
